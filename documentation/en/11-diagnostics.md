# Diagnostic Logs

The Logs tab provides a real-time view of diagnostic events generated by the app. Logs are useful for understanding app behavior, diagnosing connection issues, and reporting bugs.

---

## What Is Logged

The app logs events from the following areas:

- **Connection** — Connect, disconnect, retries
- **Polling** — Sensor reads, value filtering
- **Voice** — Wake phrase detection, speech transcription, command matching
- **App** — Startup, navigation, settings changes

---

## Severity Levels

| Level | Color | Description |
|---|---|---|
| Info | Grey / Blue | Normal events: connected, polling started, command recognized |
| Warning | Yellow / Amber | Unexpected but recoverable: retry, unrecognized voice command |
| Error | Red | Operation failed: connection lost, API error |
| Critical | Dark Red | Unrecoverable: service crashed |

---

## Categories

| Category | Events From |
|---|---|
| App | Startup, settings changes, boot events |
| DiLink | Connection status, vehicle data reads/writes |
| Polling | Sensor scheduling, timeouts, data filtering |
| Voice | Speech recognition, wake phrase, command matching |
| Automation | Rule triggers, action execution, auto-revert |
| Debug | Debug Control Panel commands and readback results |

---

## Logs Screen

The Logs screen provides filtered access to the log buffer.

**Filter bar:**

- **Row 1 — Severity chips:** All, Info, Warning, Error, Critical
- **Row 2 — Category tabs:** All, Voice, DiLink, Auto, Polling, App, Debug

Both rows are sticky at the top of the screen.

**Each log entry displays:**

1. Timestamp (HH:mm:ss)
2. Severity indicator (colored border)
3. Category badge
4. Message text

Entries are shown newest-first.

**Empty states:** The screen distinguishes between "no logs recorded yet" and "logs exist but are hidden by the active filter."

## Debug Control Panel

The Debug Control Panel is a diagnostic tool for verifying that vehicle commands actually reach the car and checking what values the car reports back. It is useful when a control (e.g., seat heat, trunk, defrost) does not seem to work — you can see whether the command was sent successfully and what the car responded with.

**How to open:** Go to **Settings > View Logs**, then tap the bug icon in the top-right corner of the Logs screen.

**What it shows:**

The panel lists every controllable action grouped by category (climate, doors, windows, sunroof, sunshade, trunk). For each action you can:

1. **Send** the command to the car
2. **Read back** the current value the car reports for that feature

**Result indicators:**

| Indicator | Meaning |
|---|---|
| **Verified (OK)** — green | Command sent and readback matches the expected value. The car accepted the command. |
| **Mismatch** — orange | Command sent but readback returned a different value. The car may not support this action or the feature ID may need an override for your model. |
| **No Readback (N/A)** — grey | Command sent but the car returned an error code. The feature may not be available on your vehicle. |
| **Failed (FAIL)** — red | The SET command itself failed to send. |

**When to use:**

- A climate or vehicle control doesn't seem to respond — use the Debug Panel to check if the command reaches the car
- You want to see the actual raw status values your car reports before sending a diagnostic report to the developer
- After a FID config update, to verify that overrides are applied correctly for your model

> [!NOTE]
> The Debug Control Panel sends real commands to the car. Use it while the vehicle is in ACC or READY state.

## Log Storage

The app keeps the last 500 entries in memory (visible in the Logs tab) and also writes daily log files to the device.

---

## Exporting Logs

Three export options are available from the **Settings** screen:

| Action | What It Does |
|---|---|
| **Clear Diagnostic Logs** | Wipes the in-memory log buffer |
| **Export Diagnostic Bundle** | Saves a comprehensive JSON file to the device containing logs, sensor data, and configuration |
| **Send Brief / Detailed Report** | Sends logs to the developer via Telegram for troubleshooting |

> [!NOTE]
> The Export Diagnostic Bundle is the best way to capture a snapshot of the app's state for bug reporting. It includes sensor data, configuration, and recent log entries all in one file.
