# Журнали діагностики

Вкладка "Журнали" надає перегляд діагностичних подій, що генеруються застосунком, у реальному часі. Журнали корисні для розуміння поведінки застосунку, діагностики проблем зі з'єднанням та повідомлення про помилки.

---

## Що записується в журнал

Застосунок реєструє події з таких областей:

- **З'єднання** — підключення, відключення, повторні спроби
- **Опитування** — зчитування сенсорів, фільтрація значень
- **Голос** — виявлення фрази пробудження, розпізнавання мовлення, зіставлення команд
- **Застосунок** — запуск, навігація, зміни налаштувань

---

## Рівні важливості

| Рівень | Колір | Опис |
|---|---|---|
| Info | Сірий / Синій | Звичайні події: підключено, опитування розпочато, команду розпізнано |
| Warning | Жовтий / Бурштиновий | Несподівані, але не критичні: повторна спроба, нерозпізнана голосова команда |
| Error | Червоний | Операцію не виконано: з'єднання втрачено, помилка API |
| Critical | Темно-червоний | Невідновна помилка: сервіс аварійно завершився |

---

## Категорії

| Категорія | Джерела подій |
|---|---|
| App | Запуск, зміни налаштувань, події завантаження |
| DiLink | Стан з'єднання, зчитування/запис даних автомобіля |
| Polling | Планування опитування сенсорів, тайм-аути, фільтрація даних |
| Voice | Розпізнавання мовлення, фраза пробудження, зіставлення команд |

---

## Екран журналів

Екран журналів надає фільтрований доступ до буфера записів.

**Панель фільтрів:**

- **Рядок 1 — Чіпи рівнів:** Усі, Info, Warning, Error, Critical
- **Рядок 2 — Чіпи категорій:** Усі, Polling, App, DiLink, Voice

Обидва рядки закріплені у верхній частині екрану.

**Кожен запис журналу відображає:**

1. Часову мітку (ГГ:хх:сс)
2. Індикатор рівня (кольорова межа)
3. Значок категорії
4. Текст повідомлення

Записи відображаються від найновіших до найстаріших.

**Порожні стани:** Екран розрізняє "журнал порожній" та "записи є, але приховані поточним фільтром".

## Панель налагодження (Debug Control Panel)

Панель налагодження — це діагностичний інструмент для перевірки, що команди керування дійсно доходять до автомобіля та зчитування фактичних значень, які автомобіль повертає. Корисно, коли елемент керування (наприклад, підігрів сидіння, багажник, обдув) начебто не працює — ви можете побачити, чи була команда успішно відправлена і що відповів автомобіль.

**Як відкрити:** Перейдіть до **Налаштування > Переглянути журнали**, потім натисніть іконку жука у верхньому правому куті екрану журналів.

**Що показує:**

Панель перераховує всі доступні дії керування, згруповані за категоріями (клімат, двері, вікна, люк, шторка, багажник). Для кожної дії можна:

1. **Надіслати** команду на автомобіль
2. **Зчитати** поточне значення, яке автомобіль повертає для цієї функції

**Індикатори результату:**

| Індикатор | Значення |
|---|---|
| **Підтверджено (OK)** — зелений | Команду надіслано, зчитане значення збігається з очікуваним. Автомобіль прийняв команду. |
| **Розбіжність (Mismatch)** — помаранчевий | Команду надіслано, але зчитане значення відрізняється. Автомобіль може не підтримувати цю дію або ідентифікатор функції потребує перевизначення для вашої моделі. |
| **Немає зчитування (N/A)** — сірий | Команду надіслано, але автомобіль повернув код помилки. Функція може бути недоступна на вашому автомобілі. |
| **Помилка (FAIL)** — червоний | Саму команду SET не вдалось надіслати. |

**Коли використовувати:**

- Елемент керування кліматом або автомобілем не реагує — використайте панель налагодження, щоб перевірити, чи доходить команда до автомобіля
- Ви хочете побачити фактичні значення статусу, які повертає ваш автомобіль, перед надсиланням діагностичного звіту розробнику
- Після оновлення FID-конфігурації — для перевірки, що перевизначення коректно застосовані для вашої моделі

> [!NOTE]
> Панель налагодження надсилає реальні команди до автомобіля. Використовуйте її, коли автомобіль у стані ACC або READY.

## Зберігання журналів

Застосунок тримає останні 500 записів у пам'яті (вони видимі на вкладці "Журнали"), а також записує щоденні лог-файли на пристрій.

---

## Експорт журналів

Три варіанти експорту доступні з екрану **Налаштування**:

| Дія | Що робить |
|---|---|
| **Очистити журнали діагностики (Clear Diagnostic Logs)** | Видаляє буфер журналів з пам'яті |
| **Експортувати діагностику (Export Diagnostic Bundle)** | Зберігає комплексний JSON-файл на пристрій, що містить журнали, дані сенсорів та конфігурацію |
| **Надіслати короткий / детальний звіт (Send Brief / Detailed Report)** | Надсилає журнали розробнику через Telegram для вирішення проблем |

> [!NOTE]
> Експорт діагностичного пакета — найкращий спосіб зафіксувати стан застосунку для повідомлення про помилки. Він містить дані сенсорів, конфігурацію та останні записи журналу — все в одному файлі.
