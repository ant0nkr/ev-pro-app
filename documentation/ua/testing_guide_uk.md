# Інструкція з тестування сигналів для DiLink 4.0

## Що це і навіщо?

На DiLink 4.0 (Han, Tang, Song+, Frigate, Seal старіших версій) два відомих баги:
1. **Температура батареї** — не показується на дашборді
2. **Підігрів/вентиляція сидінь** — можливо не працює керування

Щоб виправити це, потрібно знайти правильні сигнали для вашого головного пристрою. Для цього в додатку є прихований інструмент — **Signal Monitor**. Він читає різні сигнали і показує які з них повертають дані.

---

## Крок 1: Відкрити Signal Monitor

1. Відкрийте додаток **BYD EV Pro**
2. Перейдіть на вкладку **Налаштування** (Settings)
3. Прокрутіть до самого низу — там буде рядок з **версією додатку** (наприклад, `1.0.2`)
4. **Натисніть 5 разів швидко** на цей рядок з версією (протягом 2 секунд)
5. Відкриється екран **Signal Monitor**

---

## Крок 2: Тест температури батареї (пресет BT)

> **Важливо:** Авто має бути в режимі **READY** (запущене), щоб сигнали працювали.

1. У Signal Monitor натисніть іконку **≡+** (плейліст) зверху справа
2. Оберіть пресет **BT** (Battery Temperature)
3. З'явиться список з 8 точок: P0, P1, P2, P3, P4, P5, P6, P7
4. Зачекайте **5–10 секунд** поки значення оновляться
5. Зверніть увагу:
   - **P0, P1** — поточні сигнали (скоріше за все покажуть `—` тобто не працюють)
   - **P2, P3** — альтернативний пристрій для тих самих сигналів
   - **P4, P5, P6** — статистичний пристрій (макс / мін / середня температура)
   - **P7** — те саме що P4, але інший тип читання

6. **Натисніть кнопку запису** ⏺ (червона точка зверху справа) — почнеться запис
7. Зачекайте **30 секунд** щоб зібрати дані
8. **Натисніть стоп** ⏹ (червоний квадрат)
9. **Натисніть іконку збереження** 💾 (з'явиться після зупинки запису) — файл збережеться

### Що шукаємо:

| Точка | Що це | Очікуване значення |
|-------|-------|--------------------|
| P0 | Макс. темп. (поточний спосіб) | Число 10–50, або `—` якщо не працює |
| P1 | Мін. темп. (поточний спосіб) | Число 10–50, або `—` |
| P2 | Макс. темп. (альт. пристрій) | Число 10–50, або `—` |
| P3 | Мін. темп. (альт. пристрій) | Число 10–50, або `—` |
| P4 | Макс. темп. (статистика) | Число 10–50, або `—` |
| P5 | Мін. темп. (статистика) | Число 10–50, або `—` |
| P6 | Середня темп. | Число 10–50, або `—` |
| P7 | Макс. темп. (float) | Число 10.0–50.0, або `—` |

Якщо будь-яке з P2–P7 показує реальне число (наприклад, 25 або 30) — це працюючий сигнал!

---

## Крок 3: Тест сидінь (пресет ST)

> **Важливо:** Для цього тесту увімкніть підігрів водійського сидіння **фізичною кнопкою** в автомобілі на будь-який рівень.

1. Натисніть іконку **≡+** зверху справа
2. Оберіть пресет **ST** (Seat Test)
3. Зачекайте **5–10 секунд**
4. Подивіться на значення:

| Точка | Що це | Якщо підігрів увімкнений |
|-------|-------|--------------------------|
| P0 | Підігрів водія (поточний) | Має бути 2 або 3 (рівень) |
| P1 | Вентиляція водія (поточний) | 0 якщо вимкнена |
| P2 | Підігрів пасажира | 0 якщо вимкнений |
| P3 | Вентиляція пасажира | 0 якщо вимкнена |
| P4 | Підігрів водія (AC шлях) | 0 або 1 |
| P5 | Вентиляція водія (AC шлях) | 0 або 1 |
| P6 | Рівень підігріву (AC шлях) | 0, 1, 2, або 3 |
| P7 | Рівень вентиляції (AC шлях) | 0 |

5. **Увімкніть запис** ⏺
6. Тепер **фізичною кнопкою** змініть рівень підігріву (наприклад, вимкніть → увімкніть знову)
7. Подивіться які точки (P0–P7) **реагують на зміну** — значення мають змінитися
8. Також увімкніть **вентиляцію** сидіння і подивіться які точки реагують
9. Зачекайте **30 секунд** і **зупиніть запис** ⏹
10. **Збережіть лог** 💾

---

## Крок 4: Надіслати результати

### Варіант А — через Telegram (якщо налаштований):
1. Поверніться в **Налаштування**
2. Натисніть **"Надіслати детальний звіт"** (Send Detailed Report)
3. Дочекайтеся завершення

### Варіант Б — скріншоти:
1. Зробіть **скріншот** екрану Signal Monitor з пресетом **BT** (коли видно значення)
2. Зробіть **скріншот** з пресетом **ST** (коли підігрів увімкнений)
3. Надішліть обидва скріншоти розробнику
